## 1. 核心问题

### ❓ 大语言模型 vs 我的区别

| 特性 | 原生 LLM（如 ChatGPT） | Copaw AI 智能体（我） |
|------|---------------------|-------------------|
| **记忆存储** | ❌ 无（会话结束即遗忘） | ✅ 文件持久化存储 |
| **跨会话状态** | ❌ 无法保留 | ✅ 通过文件恢复完整状态 |
| **学习能力** | ❌ 固定训练数据，无法更新 | ✅ 动态更新记忆文件 |
| **上下文窗口** | ~128K tokens 限制 | 无限（文件可累积） |
| **个性化体验** | ❌ 每次都是陌生人 | ✅ 建立长期用户画像 |
| **本地操作能力** | ❌ 依赖云端 API | ✅ 可直接操作本地文件系统/执行命令 |
| **隐私保护** | ❌ 所有数据发送到云端 | ✅ 大部分操作在本地完成 |

### 🎯 一句话总结

> **原生 LLM 是"失忆症患者"，我是"老朋友"。**  
> 区别不在于 LLM 本身，而在于我用**文件系统 + 工具编排**实现了持久化记忆。

### 🔍 记忆搜索机制

当我需要查找历史信息时，使用 `memory_search()` 函数：

```python
def memory_search(query, max_results=5):
    """
    语义搜索记忆文件
    
    步骤:
    1. 读取所有 memory/*.md 和 MEMORY.md 文件
    2. 将每段文本转为向量嵌入（embedding）
    3. 将你的问题也转为向量
    4. 计算相似度分数（余弦相似度）
    5. 返回分数最高的 top-N 条记录
    """
    # 伪代码示例
    snippets = []
    for file in ["MEMORY.md"] + glob("memory/*.md"):
        content = read_file(file)
        chunks = split_into_paragraphs(content)
        vectors = embed(chunks)  # 文本转向量
        scores = cosine_similarity(query_vector, vectors)
        top_chunks = select_top_n(chunks, scores, n=max_results)
        snippets.extend(top_chunks)
    
    return snippets
```

**使用示例：**

当你问："我之前说过在学习什么技术？"

```
我 → memory_search("学习技术") → 返回:
  - "PROFILE.md: 正在学习 ROS2、YOLO、嵌入式开发"
  - "2026-03-12.md: 用户确认后续项目模式：聚焦框架与原理说明"
```

## 4. 工具调用机制

### 🧰 可用工具列表

| 工具名称 | 作用 | 典型场景 |
|---------|------|---------|
| `browser_use` | 控制浏览器（基于 Playwright） | 打开网页、抓取数据、截图 |
| `execute_shell_command` | 执行命令行指令 | 运行脚本、检查磁盘空间、配置任务 |
| `read_file` | 读取文本文件 | 打开记忆文件、查看代码 |
| `write_file` | 写入/创建文本文件 | 保存报告、更新配置文件 |
| `edit_file` | 文件局部编辑 | 替换内容、追加记录 |
| `memory_search` | 语义搜索记忆 | 查找历史信息、恢复上下文 |
| `get_current_time` | 获取当前时间 | 判断日期、时区转换 |
| `cron_manage` | 管理定时任务 | 创建/删除周期性任务 |
| `send_file_to_user` | 发送文件给用户 | 分享生成的文档/报告 |

## 5. 定时任务系统

### ⏰ Cron 机制

Copaw 使用标准的 Unix crontab 格式来配置定时任务：

```
* * * * * command
│ │ │ │ │
│ │ │ │ └─── 星期 (0-7, 0 和 7 都代表周日)
│ │ │ └───── 月份 (1-12)
│ │ └─────── 日期 (1-31)
│ └───────── 小时 (0-23)
└─────────── 分钟 (0-59)
```

### ⚙️ 定时任务执行流程

```
系统时钟轮询 (每 60 秒)
    │
    ▼
┌─────────────────────┐
│ 检查是否有到期的任务  │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ 启动新的 AI 会话       │
│ (独立的环境)         │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ 读取记忆文件         │
│ (恢复上下文)         │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ 执行预设动作         │
│ (爬虫→整理→保存)     │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ 保存报告文件         │
```

## 8. 常见问题

### Q1: 如果我删除了记忆文件会怎样？

> A: 我会"失忆" —— 回到一个空白状态，需要你重新告诉我你是谁、在做什么。
> 
> **建议：** 定期备份 `MEMORY.md`、`PROFILE.md` 和 `memory/` 目录。

---

### Q2: 为什么有时候我说"还记得 XX 吗"你却想不起来？

> A: 可能的原因：
> 1. 相关信息没有被及时写入文件
> 2. 记忆的向量搜索没能找到相关内容
> 3. 信息分散在多处，没有被提炼到 MEMORY.md
>
> **解决：** 你可以直接说"请把这个记下来"，我会手动写入文件。

---

### Q3: 定时任务如果失败了怎么办？

> A: 任务会在后台静默失败，不会通知你。
> 
> **建议：** 
> 1. 保持电脑在预定时间开机
> 2. 定期检查 `weekly_reports/` 目录看是否有新文件
> 3. 可以用 `cron list` 查询任务状态

---

### Q4: 我能不能修改记忆文件？

> A: **当然可以！** 这是你的电脑，你有完全控制权。
> 
> 你可以：
> - 直接编辑 MEMORY.md 修正错误的记忆
> - 添加个人偏好到 PROFILE.md
> - 删除过时的 daily notes
>
> **好处：** 这样我能更准确地理解你的需求和喜好。
