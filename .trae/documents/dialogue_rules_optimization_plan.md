# 对话规则优化实施计划

## 1. 项目概述

本计划旨在优化智能体的对话规则，提升响应策略和用户互动体验。主要包含问题分类处理机制和全场景开放性提问要求两个核心部分。

## 2. 任务分解与优先级

### [x] 任务 1: 增强问题分类处理机制
- **优先级**: P0
- **依赖项**: None
- **描述**:
  - 优化 `meta_router.py` 中的决策逻辑
  - 实现无法直接回答问题的分类判断流程
  - 区分闲聊类问题和非闲聊类问题
  - 为闲聊类问题设置专门的处理路径
- **成功标准**:
  - 系统能够准确区分闲聊类问题和非闲聊类问题
  - 闲聊类问题能够自动触发闲聊模式回答
  - 非闲聊类问题能够正确执行知识边界响应流程
- **测试要求**:
  - `programmatic` TR-1.1: 系统能够正确识别至少10个不同类型的闲聊问题
  - `programmatic` TR-1.2: 系统能够正确识别至少10个不同类型的非闲聊问题
  - `human-judgement` TR-1.3: 分类判断逻辑准确，无明显误判
- **注意事项**:
  - 确保分类逻辑清晰明确
  - 避免将任务型问题误判为闲聊
  - 保持分类判断的准确性和一致性

### [x] 任务 2: 实现全场景开放性提问机制
- **优先级**: P0
- **依赖项**: 任务 1
- **描述**:
  - 在所有回答后附加与用户问题主题相关的开放性提问
  - 设计开放性提问的生成逻辑
  - 确保提问自然融入对话，避免生硬突兀
- **成功标准**:
  - 所有回答后都能附加相关的开放性提问
  - 开放性提问满足与主题相关、邀请式语气、引导对话深入的要求
  - 开放性提问避免简单的封闭式问题
- **测试要求**:
  - `programmatic` TR-2.1: 系统能够为至少15个不同类型的问题生成合适的开放性提问
  - `human-judgement` TR-2.2: 开放性提问自然融入对话，无生硬感
  - `human-judgement` TR-2.3: 开放性提问与问题主题相关，能够引导对话继续
- **注意事项**:
  - 开放性提问需与问题主题相关
  - 采用邀请式、探索式或建议式语气
  - 避免简单的封闭式问题

### [x] 任务 3: 优化闲聊模式响应
- **优先级**: P1
- **依赖项**: 任务 1
- **描述**:
  - 增强 `small_talk_tool.py` 中的闲聊响应能力
  - 确保闲聊回答自然、流畅、符合人设
  - 为不同类型的闲聊问题提供多样化的回应
- **成功标准**:
  - 闲聊模式能够生成自然、流畅的回答
  - 回答符合智能体的人设和语言风格
  - 不同类型的闲聊问题能够得到多样化的回应
- **测试要求**:
  - `programmatic` TR-3.1: 系统能够为至少15个不同类型的闲聊问题生成合适的回答
  - `human-judgement` TR-3.2: 闲聊回答自然、流畅、符合人设
  - `human-judgement` TR-3.3: 不同类型的闲聊问题能够得到多样化的回应
- **注意事项**:
  - 保持回答的一致性和连贯性
  - 避免重复回答
  - 确保回答符合智能体的人设

### [x] 任务 4: 测试与验证
- **优先级**: P1
- **依赖项**: 任务 1, 任务 2, 任务 3
- **描述**:

  - 测试问题分类准确性
  - 测试开放性提问的质量和相关性
  - 测试闲聊模式的响应质量

  - 系统能够正确处理各种类型的问题
  - 开放性提问质量高，与问题主题相关
  - 闲聊模式能够生成自然、流畅的回答
- **测试要求**:
  - `programmatic` TR-4.1: 系统能够正确处理至少30个不同类型的问题
  - `human-judgement` TR-4.2: 整体响应质量高，互动性强
  - `human-judgement` TR-4.3: 系统能够保持专业、帮助性的同时增强互动性
- **注意事项**:
  - 测试覆盖各种类型的问题
  - 测试不同难度级别的问题
  - 测试不同回答复杂度的情况

4. 为非闲聊类问题设置知识边界响应流程

### 3.2 全场景开放性提问机制

2. 实现开放性提问的生成逻辑
3. 确保开放性提问与问题主题相关
4. 确保开放性提问自然融入对话

### 3.3 闲聊模式响应优化

修改 `src/skills/mcp_skills/small_talk_tool.py` 中的闲聊响应逻辑，增强闲聊回答的质量和多样性。具体实现：

1. 增加更多类型的闲聊问题识别
2. 为不同类型的闲聊问题提供更多样化的回应
3. 确保闲聊回答符合智能体的人设和语言风格

## 6. 风险评估

1. **分类准确性风险**:
   - 可能出现问题分类不准确的情况
   - 应对措施：不断优化分类逻辑，增加更多的训练数据

2. **开放性提问质量风险**:
   - 可能生成与问题主题不相关的开放性提问
   - 应对措施：设计更合理的开放性提问模板，增加主题相关性检查

3. **闲聊模式响应质量风险**:
   - 可能生成不自然、不流畅的闲聊回答
   - 应对措施：不断优化闲聊响应逻辑，增加更多样化的回应

4. **系统稳定性风险**:
   - 可能因修改而影响系统的稳定性
   - 应对措施：进行充分的测试，确保系统的稳定性和可靠性
