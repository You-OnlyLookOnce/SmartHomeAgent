# 本地天气工具使用说明

## 1. 概述

本地天气工具是一个使用和风天气API实现的天气获取工具，用于替代原来的远程MCP天气服务。它具有以下特点：

- **本地化**: 直接调用和风天气API，不依赖远程MCP服务
- **多功能**: 支持获取实时天气和7天天气预报
- **详细信息**: 提供温度、天气状况、湿度、风力等详细信息
- **中文支持**: 完全中文天气描述
- **可靠性高**: 直接对接官方API，响应速度快

## 2. 实现原理

本地天气工具使用Python的httpx库直接调用和风天气API：

- **位置查询**: 使用城市名称查询位置ID
- **实时天气**: 获取指定位置的实时天气数据
- **天气预报**: 获取指定位置的7天天气预报
- **数据格式化**: 将API返回的数据格式化为友好的中文信息

## 3. 集成方式

本地天气工具已集成到系统的资源注册表中，通过以下方式调用：

### 3.1 直接调用

```python
from src.skills.mcp_skills.simple_local_weather_tool import simple_local_weather_tool

# 获取北京的天气
result = await simple_local_weather_tool.get_weather("北京")

# 获取上海的天气
result = await simple_local_weather_tool.get_weather("上海")
```

### 3.2 通过资源注册表调用

```python
from src.core.resource_registry import registry

# 获取天气工具
weather_tool = registry.get_tool("get_weather")

# 调用天气工具
result = await weather_tool("广州")
```

### 3.3 通过智能决策系统调用

系统的智能决策系统会自动识别天气相关的问题，并调用本地天气工具：

- "北京的天气怎么样？"
- "上海今天天气如何？"
- "广州的温度是多少？"

## 4. 配置要求

本地天气工具需要配置和风天气API密钥：

### 4.1 环境变量配置

在 `.env` 文件中添加以下配置：

```
# 和风天气API配置
HEFENG_WEATHER_API_KEY=your_api_key_here
```

### 4.2 API密钥获取

1. 访问 [和风天气开发者平台](https://dev.qweather.com/)
2. 注册并登录账号
3. 创建应用，获取API密钥
4. 将API密钥配置到环境变量中

## 5. 支持的功能

本地天气工具支持以下功能：

- **实时天气**: 获取当前天气状况、温度、体感温度、湿度、风力等信息
- **7天天气预报**: 获取未来7天的天气状况、最高温度、最低温度、风力等信息
- **位置查询**: 支持通过城市名称查询位置信息
- **错误处理**: 处理API调用失败、位置不存在等错误情况

## 6. 输出格式

天气工具的输出格式为：

```
城市名称的实时天气：
- 天气状况: 晴
- 温度: 25°C
- 体感温度: 26°C
- 湿度: 60%
- 风力: 东南风 3级
- 气压: 1013hPa
- 能见度: 10km
- 更新时间: 2026-03-23 12:00:00

7天天气预报：
2026-03-23: 晴 / 多云，温度 18°C ~ 25°C，东南风 3级
2026-03-24: 多云 / 阴，温度 17°C ~ 24°C，南风 2级
...
```

## 7. 错误处理

本地天气工具具有完善的错误处理机制：

- **未配置API密钥**: 当未配置和风天气API密钥时，返回友好的错误信息
- **位置不存在**: 当查询的位置不存在时，返回友好的错误信息
- **API调用失败**: 当API调用失败时，返回友好的错误信息
- **其他错误**: 捕获并记录其他错误，返回友好的错误信息

## 8. 性能优化

- **异步执行**: 使用asyncio和httpx进行异步API调用，不阻塞主线程
- **超时处理**: 设置API调用超时，避免长时间等待
- **错误重试**: 对API调用失败的情况进行适当的错误处理

## 9. 与原有系统的兼容性

本地天气工具完全兼容原有系统的接口：

- 保持与原有MCP天气工具相同的方法签名
- 保持相同的返回格式
- 集成到相同的资源注册表中

## 10. 测试验证

本地天气工具已通过以下测试：

- 未配置API密钥的错误处理
- 不同城市的天气获取
- 无效城市的错误处理
- 空城市名的错误处理

## 11. 总结

本地天气工具是一个可靠、高效的天气获取解决方案，它：

- 替代了依赖远程MCP服务的天气工具
- 直接对接和风天气API，提供更详细的天气信息
- 支持实时天气和7天天气预报
- 与现有系统完全兼容

通过使用本地天气工具，系统不再依赖外部MCP服务，提高了系统的可靠性和响应速度，同时提供了更丰富的天气信息。
