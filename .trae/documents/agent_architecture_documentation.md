# 智能体架构文档

## 1. 架构概述

当前智能体架构采用模块化设计，基于LLM驱动的决策系统，结合了先进的记忆管理和人格化处理技术，实现了一个高度智能、个性化的智能家居助手。

## 2. 核心模块

### 2.1 决策系统

**MetaCognitionRouterV2** - 基于LLM Function Calling的新一代决策引擎：
- **功能**：完全由LLM自主决策
- **工作流程**：
  1. 接收用户输入
  2. 调用LLM Function Calling决定使用哪些工具
  3. 执行工具并获取结果
  4. 生成最终回复
- **技术特点**：
  - 集成了七牛云实时推理API
  - 支持流式响应
  - 完整的错误处理机制

### 2.2 记忆系统

**MemorySystem** - 包含四个独立记忆模块的全新架构：

1. **工作记忆模块**：
   - 功能：作为ReAct循环的"大脑临时记事本"
   - 特点：记录当前任务目标、实时更新执行进度及中间状态信息，支持动态修改与实时查询

2. **对话记忆模块**：
   - 功能：作为上下文理解的"聊天记录册"
   - 特点：完整存储用户与智能体的所有对话内容，为LLM提供完整上下文信息

3. **执行记忆模块**：
   - 功能：作为MCP工具调用的"操作日志本"
   - 特点：详细记录每一步MCP工具调用的完整信息（包括调用参数、返回结果、执行状态等）

4. **长期记忆模块**：
   - 功能：作为个性化智能体的"用户档案+知识库"
   - 特点：存储用户偏好、历史交互模式、设备信息及领域知识等长期数据

### 2.3 人格系统

**Personality Architecture** - 模块化的人格管理系统：

1. **永久人格核心模块**：
   - 功能：负责智能体基础人设、沟通风格和行为禁忌的定义与维护
   - 特点：采用不可变数据结构存储核心人格参数，防止运行时意外修改

2. **输出人格化润色器**：
   - 功能：对LLM生成的原始输出内容进行人格化转换
   - 特点：确保转换后的输出内容符合"悦悦"人格特征

### 2.4 工具系统

**MCP (Module Communication Protocol) Tool System**：
- **功能**：将所有决策类型/资源统一改造为MCP工具模块
- **工具类型**：
  - 文件操作工具
  - 设备控制工具
  - 网络搜索工具
- **技术特点**：
  - 统一的工具注册和管理机制
  - 支持异步工具执行
  - 完整的错误处理

## 3. 技术方案

### 3.1 LLM集成

- **API集成**：集成七牛云实时推理API，支持Function Calling
- **模型选择**：使用kimi2.5模型
- **调用方式**：异步调用，支持流式响应

### 3.2 记忆管理

- **存储方式**：
  - 工作记忆、对话记忆、执行记忆：内存存储
  - 长期记忆：文件存储（JSON格式）
- **记忆整合**：在ReAct循环中，从四个记忆模块中读取全部相关信息，按照指定格式拼接后作为上下文提供给LLM
- **记忆更新**：实时更新各记忆模块，确保信息的及时性和准确性

### 3.3 人格化处理

- **人格定义**：使用不可变数据结构定义"悦悦"的核心人格参数
- **润色机制**：对LLM生成的原始输出进行多步骤处理：
  1. 基础文本处理
  2. 注入悦悦风格（自称、关心问候、征询意见）
  3. 添加适当的emoji
  4. 调整语气和表达方式
  5. 确保符合行为禁忌
- **性能优化**：确保润色处理延迟控制在100ms以内

### 3.4 系统集成

- **Web框架**：使用FastAPI构建RESTful API
- **异步处理**：全面采用异步编程模型，提高系统响应速度
- **错误处理**：完善的错误处理机制，确保系统稳定性
- **日志管理**：详细的日志记录，便于问题排查

## 4. 创新点

### 4.1 记忆系统创新

- **四模块记忆架构**：将记忆系统分为工作记忆、对话记忆、执行记忆和长期记忆四个独立模块，实现了记忆的分层管理和高效检索
- **实时记忆整合**：在ReAct循环中实时整合四个记忆模块的信息，为LLM提供完整的上下文
- **记忆持久化**：长期记忆模块实现了数据的持久化存储，确保用户偏好和历史交互模式的保存

### 4.2 人格系统创新

- **不可变人格核心**：采用不可变数据结构存储核心人格参数，确保人格特征的稳定性和一致性
- **实时人格化润色**：在输出流程的最终环节对LLM生成的内容进行人格化转换，确保输出符合"悦悦"的人格特征
- **性能优化**：通过优化算法和数据结构，确保人格化润色处理延迟控制在100ms以内

### 4.3 决策系统创新

- **LLM驱动决策**：完全移除人工规则，由LLM通过Function Calling自主决策，实现了真正的AI驱动决策
- **工具化改造**：将所有决策类型/资源统一改造为MCP工具模块，提高了系统的可扩展性和可维护性
- **流式响应**：支持流式响应，提高用户体验的实时性

### 4.4 系统架构创新

- **模块化设计**：采用高度模块化的设计，各模块职责明确，便于维护和扩展
- **异步编程**：全面采用异步编程模型，提高系统的并发处理能力和响应速度

## 5. 系统运行流程

### 5.1 启动初始化阶段

1. 加载长期记忆模块中的用户偏好设置与设备库信息
2. 初始化工作记忆、对话记忆及执行记忆模块为空状态
3. 初始化永久人格核心模块，与长期记忆模块一同完成初始化流程
4. 初始化输出人格化润色器
5. 注册所有MCP工具

### 5.2 用户输入处理阶段

1. 接收用户输入
2. 将用户输入内容完整写入对话记忆模块
3. 从四个记忆模块中读取全部相关信息，按照指定格式拼接后作为上下文提供给LLM
4. 调用LLM Function Calling决定使用哪些工具
5. 执行工具并获取结果
6. 将工具执行结果写入执行记忆模块
7. 根据工具执行结果实时更新工作记忆模块中的任务进度信息
8. 调用LLM生成最终回复
9. 对回复内容进行人格化润色
10. 将润色后的回复内容写入对话记忆模块
11. 返回最终回复给用户
