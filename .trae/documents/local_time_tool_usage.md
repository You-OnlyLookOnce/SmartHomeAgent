# 本地时间工具使用说明

## 1. 概述

本地时间工具是一个使用 Python 内置 `datetime` 模块实现的时间获取工具，用于替代原来的远程 MCP 时间服务。它具有以下特点：

- **本地化**: 不依赖远程 MCP 服务，直接在本地计算时间
- **多时区支持**: 支持全球各个时区的时间获取
- **夏令时检测**: 自动检测并显示夏令时信息
- **可靠性高**: 不受网络连接影响，响应速度快

## 2. 实现原理

本地时间工具使用 Python 标准库中的 `datetime` 模块和 `zoneinfo` 模块来获取和处理时间信息：

- `datetime` 模块: 提供时间获取和格式化功能
- `zoneinfo` 模块: 提供时区支持，确保时区转换的准确性

## 3. 集成方式

本地时间工具已集成到系统的资源注册表中，通过以下方式调用：

### 3.1 直接调用

```python
from src.skills.mcp_skills.simple_local_time_tool import simple_local_time_tool

# 获取默认时区（Asia/Shanghai）的时间
result = await simple_local_time_tool.get_time()

# 获取指定时区的时间
result = await simple_local_time_tool.get_time("America/New_York")
```

### 3.2 通过资源注册表调用

```python
from src.core.resource_registry import registry

# 获取时间工具
time_tool = registry.get_tool("get_local_time")

# 调用时间工具
result = await time_tool()
```

### 3.3 通过智能决策系统调用

系统的智能决策系统会自动识别时间相关的问题，并调用本地时间工具：

- "现在几点了？"
- "当前时间"
- "现在的日期和时间"

## 4. 支持的时区

本地时间工具支持所有 IANA 时区，常见的时区包括：

- `Asia/Shanghai` (中国标准时间)
- `America/New_York` (美国东部时间)
- `Europe/London` (英国标准时间)
- `Asia/Tokyo` (日本标准时间)
- `Australia/Sydney` (澳大利亚东部时间)

## 5. 输出格式

时间工具的输出格式为：

```
当前时间（时区）: 时间戳，星期几，时间类型
```

例如：

```
当前时间（Asia/Shanghai）: 2026-03-23T20:18:31+08:00，Monday，标准时间
```

## 6. 错误处理

本地时间工具具有完善的错误处理机制：

- 无效时区: 当提供的时区无效时，会自动使用本地时区
- 其他错误: 会捕获并记录错误，返回友好的错误信息

## 7. 性能优化

- **缓存机制**: 避免重复的时区初始化
- **异步执行**: 支持异步调用，不阻塞主线程
- **高效计算**: 直接使用系统时间，无需网络请求

## 8. 与原有系统的兼容性

本地时间工具完全兼容原有系统的接口：

- 保持与原有 MCP 时间工具相同的方法签名
- 保持相同的返回格式
- 集成到相同的资源注册表中

## 9. 测试验证

本地时间工具已通过以下测试：

- 默认时区时间获取
- 不同时区时间获取

## 10. 总结

本地时间工具是一个可靠、高效的时间获取解决方案，它：

- 替代了依赖远程 MCP 服务的时间工具
- 提供了更快速、更可靠的时间获取能力
- 支持多时区和夏令时检测
- 与现有系统完全兼容

通过使用本地时间工具，系统不再依赖外部 MCP 服务，提高了系统的可靠性和响应速度。
