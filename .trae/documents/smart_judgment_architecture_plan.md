# 智能判断架构实现计划

## 项目背景
根据文档 `YUEYUE/docs/HOW_TO_MAKE_YUEYUE_SMARTER.md`，我们需要在现有智能体架构的基础上实现一个智能判断系统，让智能体能够自己判断使用什么方式回答用户问题，而不是盲目依赖搜索工具。

## 现有架构分析

### 核心组件
1. **API网关** (`src/gateway/api_gateway.py`) - 处理HTTP请求，是系统的入口点
2. **会话管理器** (`src/agent/independent_session_manager.py`) - 管理对话会话和历史
3. **多层决策系统** (`src/skills/decision_skills/multi_layer_decision.py`) - 分析用户查询
4. **网络搜索** (`src/skills/search_skills/web_search.py`) - 执行网络搜索
5. **LLM调用** (`src/ai/qiniu_llm.py`) - 调用语言模型
6. **内置知识库** (`src/skills/knowledge_skills/built_in_knowledge.py`) - 存储常见问题的回答

### 现有流程
1. 用户发送消息到 `/api/chat` 接口
2. API网关提取用户输入和会话信息
3. 多层决策系统分析查询类型
4. 根据分析结果，选择从知识库回答、执行搜索或直接调用LLM
5. 保存对话历史并返回响应

## 新架构设计

### 设计理念
- **不处理具体问题，给智能体「学会解决问题的方法」**
- **保持与现有功能完全兼容**
- **最小化对现有代码的修改**
- **采用模块化设计，易于扩展**

### 新组件

1. **资源注册表** (`src/core/resource_registry.py`)
   - 管理所有可用能力（工具、API、知识库等）
   - 为LLM提供资源清单，帮助其做出决策

2. **元认知路由器** (`src/core/meta_router.py`)
   - 让LLM自己判断用什么方式回答问题
   - 根据用户输入和资源清单做出决策
   - 执行相应的行动（调用工具、搜索、承认不知道等）

3. **工具管理器** (`src/tools/tool_manager.py`)
   - 管理和执行各种工具函数
   - 提供统一的接口来调用不同类型的工具

### 集成点
- **API网关** - 在处理请求时使用元认知路由器
- **多层决策系统** - 作为资源注册表的一部分，提供决策能力
- **LLM调用** - 用于元认知决策过程

## 验收标准

完成以下测试即表示实现成功:

| 测试项 | 预期结果 |
|--------|----------|
| `今天是几号？` | 返回真实日期，不触发搜索 |
| `现在几点？` | 返回真实时间，不触发搜索 |
| `一加一等于几？` | 计算正确，不触发搜索 |
| `你是谁？` | 返回身份信息，不触发搜索 |
| `北京明天天气` | 触发搜索/API |
| `完全陌生的问题` | 诚实承认 + 邀请补充 |
| 现有功能 | 保持正常工作 |

## 核心原则

1. **不写死答案** - 用系统API动态获取实时数据
2. **不给固定规则** - 让LLM根据资源清单自己做选择
3. **不修改核心逻辑来加新功能** - 只用资源注册机制扩展
4. **保持与现有功能兼容** - 确保现有系统不受影响

## 实现注意事项

- **兼容性**: 确保与现有功能完全兼容，不破坏现有代码
- **可扩展性**: 设计模块化架构，易于添加新资源和工具
- **性能**: 确保决策过程快速高效，不影响用户体验
- **可靠性**: 确保工具调用和决策执行的可靠性
- **可维护性**: 代码结构清晰，易于理解和维护

通过实现这个智能判断架构，我们的智能体将能够更智能地处理各种类型的问题，提高回答的准确性和效率，同时保持与现有功能的完全兼容。
