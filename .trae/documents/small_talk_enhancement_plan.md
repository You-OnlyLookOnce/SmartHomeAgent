# 智能闲聊检测与响应机制增强计划

## 1. 项目概述

本计划旨在增强当前系统，添加智能闲聊（small talk）检测和响应机制，使系统能够准确识别一般性的闲聊话题，并生成自然、流畅的闲聊回复，同时区分任务导向型查询，确保系统对不同类型的输入提供适当的响应。

## 2. 实现计划

### [x] 任务 1: 创建闲聊检测与响应工具
- **优先级**: P0
- **依赖项**: 无
- **描述**:
  - 创建一个新的工具模块，实现闲聊话题的检测和响应生成
  - 设计闲聊话题分类模型，能够识别常见的闲聊主题
  - 实现闲聊响应生成功能，确保回复自然、流畅
- **成功标准**:
  - 工具能够准确识别闲聊话题
  - 生成的闲聊回复符合系统人设，自然流畅
- **测试要求**:
  - `programmatic` TR-1.1: 工具能够正确识别至少80%的闲聊话题
  - `human-judgement` TR-1.2: 生成的闲聊回复自然、符合上下文、符合系统人设
- **注意事项**:
  - 闲聊检测应考虑上下文，不仅仅是关键词匹配
  - 回复应符合系统的整体人设和语言风格

## 3. 技术实现细节

### 3.1 闲聊检测与响应工具

创建一个新的工具模块 `src/skills/mcp_skills/small_talk_tool.py`，实现以下功能：

- 闲聊话题检测：基于规则和关键词识别闲聊话题
- 闲聊话题分类：将闲聊话题分为不同类别（如问候、情感表达、日常对话等）
- 闲聊响应生成：根据话题类别和上下文生成自然、流畅的回复

### 3.2 资源注册

在 `src/core/resource_registry.py` 中注册闲聊资源：

- 资源名称：`small_talk`
- 资源描述：处理一般性的闲聊话题，如问候、情感表达、日常对话等
- 工具函数：调用闲聊检测与响应工具
- 示例：提供常见的闲聊话题示例

### 3.3 MetaCognitionRouter增强

修改 `src/core/meta_router.py` 中的 `decide` 方法：

- 增强决策逻辑，添加对闲聊话题的识别
- 为闲聊话题选择 `small_talk` 资源
- 确保任务导向型查询仍然选择适当的功能资源

### 3.4 闲聊话题分类模型

实现一个基于规则和关键词的分类模型，涵盖以下类别：

- 问候语（如"你好"、"早上好"等）
- 情感表达（如"我很高兴"、"我很伤心"等）
- 日常对话（如"今天过得怎么样"、"最近忙什么呢"等）
- 兴趣爱好（如"你喜欢什么电影"、"你喜欢什么音乐"等）
- 天气相关（如"今天天气真好"、"下雨了"等）

### 3.5 闲聊响应生成机制

基于以下因素生成闲聊回复：

- 闲聊话题类别
- 对话历史和上下文
- 系统人设和语言风格
- 情感状态（如果用户表达了情感）
