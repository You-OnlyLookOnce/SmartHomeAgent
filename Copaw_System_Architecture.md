## 1. 核心设计哲学

### 🎯 三大支柱

| 支柱 | 含义 | 实现方式 |
|------|------|---------|
| **会话隔离** | 每个聊天窗口拥有独立上下文 | UUID + 内存空间隔离 |
| **记忆共享** | 全局知识对所有窗口可见 | 文件系统的持久化存储 |
| **动态继承** | 新窗口自动继承用户画像 | 启动时加载全局记忆文件 |

### 💡 设计理念

```
原生 LLM = 单线程大脑（无状态）
         → 每次对话都是全新开始

Copaw AI = 多任务操作系统
         → 每个窗口是一个"进程"
         → 全局记忆是"共享内存"
         → 文件系统是"磁盘存储"
```

## 4. 记忆隔离与共享策略

### 🧩 记忆分层模型

```
┌───────────────────────────────────────────────────────────────┐
│                    记忆层次结构                                 │
├───────────────────────────────────────────────────────────────┤
│                                                                │
│  层级 1: 全局记忆 (所有窗口共享)                                │
│  ├── MEMORY.md       (长期记忆精华)                            │
│  ├── PROFILE.md      (用户画像)                                │
│  ├── AGENTS.md       (行为准则)                                │
│  ├── SOUL.md         (灵魂设定)                                │
│  └── HEARTBEAT.md    (待办提醒)                                │
│                                                                │
│  特点：                                                         │
│  - 任何窗口启动时都会读取                                       │
│  - 任何窗口都可以更新（需要权限控制）                            │
│  - 删除某个窗口不会影响这些数据                                 │
│                                                                │
│  ───────────────────────────────────────────────────────────  │
│                                                                │
│  层级 2: 会话级日志 (跨窗口混合记录)                             │
│  └── memory/YYYY-MM-DD.md                                     │
│                                                                │
│  特点：                                                         │
│  - 按日期组织，不按窗口分割                                     │
│  - 所有窗口的日常事件都写在这里                                 │
│  - 可通过 memory_search() 语义搜索找到                        │
│                                                                │
│  ───────────────────────────────────────────────────────────  │
│                                                                │
│  层级 3: 会话局部上下文 (严格隔离)                               │
│  ├── Session A 的 conversation_history                         │
│  ├── Session B 的 conversation_history                         │
│  └── Session C 的 conversation_history                         │
│                                                                │
│  特点：                                                         │
│  - 完全内存驻留（或按需加载）                                   │
│  - 窗口A无法访问窗口B的历史消息                                 │
│  - 删除窗口后该数据立即清除                                     │
│                                                                │
└───────────────────────────────────────────────────────────────┘
```

### 🔍 记忆访问规则矩阵

| 数据类型 | 窗口A可见？ | 窗口B可见？ | 删除窗口A后保留？ | 更新频率 |
|---------|------------|------------|------------------|---------|
| `MEMORY.md` | ✅ | ✅ | ✅ | 定期提炼 |
| `PROFILE.md` | ✅ | ✅ | ✅ | 手动/自动 |
| `memory/2026-03-15.md` | ✅ | ✅ | ✅ | 实时追加 |
| Session A 历史 | ❌ | ❌ | ❌ | 实时追加 |
| Session B 历史 | ❌ | ❌ | ❌ | 实时追加 |

---

### 💡 为什么这样设计？

#### 场景模拟

**场景 1：你在窗口 A 讨论 ROS2 学习**
```
你："我想问问你如何做西红柿炒鸡蛋"
我："好的！让我帮你调查下......"
   → 写入 memory/2026-03-15.md: "用户计划在窗口A进行西红柿炒鸡蛋做法的讨论"
   → 提炼到 MEMORY.md: "用户对做饭感兴趣"
```

**场景 2：你切换到窗口 B 讨论智能家居**
```
你："教我如何做红烧肉"
我："没问题！根据你先前做过西红柿炒鸡蛋的背景，可以考虑..."
   → 读取 MEMORY.md 知道你有做过饭的背景而不是零基础
   → 但不知道窗口 A 的具体讨论细节（除非在 memory/*.md 里）
```

**场景 3：删除窗口 A**
```
操作：删除"智能家居"窗口
结果：
  ✅ MEMORY.md 依然保留"用户对做饭感兴趣"
  ✅ memory/2026-03-15.md 依然有原始记录
  ❌ 窗口 A 的详细对话历史被清除
```

### 💾 持久化策略对比

| 数据类型 | 存储位置 | 写入时机 | 读取时机 | 清理策略 |
|---------|---------|---------|---------|---------|
| 会话列表 | `chats.json` | 创建/重命名/删除 | 启动时/切换时 | 手动删除条目 |
| 全局记忆 | `MEMORY.md` 等 | 自动提炼/手动更新 | 每次启动 | 永不自动删除 |
| 每日日志 | `memory/*.md` | 实时追加 | 启动时/搜索时 | 定期归档/手动清理 |
| 会话历史 | 内存 + `sessions/` | 每轮对话后 | 恢复会话时 | 删除窗口时清空 |
| 工具缓存 | `tool_result/` | 执行后 | 必要时 | TTL 过期/手动清理 |

### 🔄 自动摘要机制（Memory Compaction）

当会话历史过长时，系统会自动触发摘要：

```python
def compact_session_memory(session_id):
    """
    将过长的对话历史提炼成精华，写入 MEMORY.md
    """
    history = get_conversation_history(session_id)
    
    if len(history) > MAX_HISTORY_LENGTH:
        # 1. 提取关键点
        key_insights = extract_key_points(history)
        
        # 2. 更新全局记忆
        update_memory_md(key_insights)
        
        # 3. 截断本地历史，只保留最近 N 条
        keep_recent_n(history, n=KEEP_RECENT_N)
        
        # 4. 将旧历史归档到 file
        archive_to_file(session_id, old_history)
```

**参数示例（来自 config.json）：**
```json
{
  "agents": {
    "running": {
      "memory_compact_ratio": 0.75,
      "memory_reserve_ratio": 0.1,
      "enable_tool_result_compact": false
    }
  }
}
```

## 6. 会话生命周期管理

### 🔄 完整生命周期图

```
┌──────────┐      ┌──────────┐      ┌──────────┐      ┌──────────┐
│  Created │─────►│  Active  │─────►│  Archived│─────►│  Deleted │
└──────────┘      └──────────┘      └──────────┘      └──────────┘
      ▲                 │                 │                │
      │                 ▼                 ▼                │
      │          ┌──────────┐      ┌──────────┐           │
      └──────────│  Paused  │◄────►│  Compact │◄──────────┘
                 └──────────┘      └──────────┘
```

### 📋 状态转换详解

#### 1️⃣ **Created（创建）**
- 生成新的 `session_id`
- 初始化空的历史记录
- 加载全局记忆文件
- 注册到 `chats.json`

#### 2️⃣ **Active（活跃）**
- 接收用户消息
- 执行工具调用
- 实时更新历史记录
- 可能触发自动摘要

#### 3️⃣ **Paused（暂停）**
- 用户关闭窗口但未删除
- 上下文保留在内存或 `sessions/` 目录
- 可快速恢复

#### 4️⃣ **Archived（归档）**
- 历史超过阈值，自动压缩
- 精华提炼到 `MEMORY.md`
- 原始内容存档到 `sessions/{session_id}_archive.json`

#### 5️⃣ **Deleted（删除）**
- 从 `chats.json` 移除条目
- 清除内存中的上下文
- （可选）清理 `sessions/` 中的缓存文件
- **不影响** `MEMORY.md` 和 `memory/*.md`

### 🛠️ 自定义扩展点

| 扩展点 | 修改位置 | 示例 |
|--------|---------|------|
| **全局记忆文件** | `.copaw/MEMORY.md` 等 | 添加新项目背景 |

## 🎯 总结

| 特性 | 实现原理 | 用户体验 |
|------|---------|---------|
| **多窗口隔离** | 不同的 `session_id` 对应不同的内存上下文 | 每个窗口独立对话，互不干扰 |
| **全局记忆** | 共享 `MEMORY.md`、`PROFILE.md` 等文件 | 所有窗口都知道你的背景和喜好 |
| **重命名** | 修改 `chats.json[name]` 字段 | 灵活管理窗口标签 |
| **删除窗口** | 从 `chats.json` 移除条目，保留全局记忆 | 清理垃圾但不丢失重要信息 |
| **自动摘要** | 定期将长对话提炼到 `MEMORY.md` | 长期记忆持续增长 |
| **语义搜索** | 向量嵌入 + 余弦相似度 | 快速找到过去的任何信息 |

> **"像人类一样思考"** —— 我们记得所有认识的人（全局记忆），但对每个人有不同的相处方式（局部上下文）。  
> 
> 这套架构让你能够：
> - ✅ 同时处理多个独立任务
> - ✅ 保持对用户的长期了解
> - ✅ 灵活管理对话历史
> - ✅ 保护隐私（不需要的信息随时删除）
